# 《College》UI 结构与素材规格

> **文档定位**：统一当前 UI 分层结构、场景状态、接线现状与素材出图规格。  
> **核心原则**：区分“当前默认玩家入口”“保留中的旧 UI 回退路径”“仍可继续增强的原型能力”。  
> **最后核对**：2026-04-08

---

## 1. 当前 UI 结论

### 1.1 当前准确说法

当前 UI 状态应表述为：

- 默认运行入口已经切到 `scenes/ui/ui_test_v2.tscn`
- V2 UI 已经接上现行 core，能作为日常启动入口
- 旧 `ui_test_root` 仍保留，作为冻结回退路径
- `scenes/test_scene.tscn` 仍保留，作为 21 天自动测试入口
- 手机、骰子、剧情、日结算都已经有可用交互，但仍可继续增强内容深度
- 顶部状态栏右侧已接入电脑与手机两个功能入口：电脑为待开发占位，手机直连现有 PhoneLayer

### 1.2 当前不要再写成的说法

以下表述当前都不准确：

- “默认入口还是 `scenes/test_scene.tscn`”
- “`ui_test_v2` 只是展示原型”
- “手机只能靠关闭按钮关掉”
- “剧情层只能靠键盘推进”

## 2. 当前 UI 场景结构

### 2.1 现存主要 UI 场景

工程内当前与 UI 直接相关的主场景有三类：

| 场景 | 当前定位 |
|---|---|
| `scenes/ui/ui_test_v2.tscn` | 当前默认玩家入口 / V2 UI 主控链 |
| `scenes/ui/ui_test_root.tscn` | 冻结保留的旧版整合型 UI Root |
| `scenes/main.tscn` | 旧 UI 包装入口 |

### 2.2 当前推荐的日常启动方式

平时继续启动项目时，直接运行默认入口即可：

- `project.godot` → `run/main_scene="res://scenes/ui/ui_test_v2.tscn"`

如需做核心数值回归验证，则手动运行：

- `scenes/test_scene.tscn`

## 3. V2 UI 主链现状

### 3.1 `scenes/ui/ui_test_v2.tscn`

这一套现在是当前玩家可直接进入的 V2 主界面，包含：

- `TodayView`
- `StoryLayer`
- `DiceLayer`
- `PhoneLayer`
- `DaySummaryLayer`
- `TestControls`

### 3.2 `scenes/ui/ui_test_v2.gd`

这一层现在不是假数据舞台，而是当前 V2 UI 的主控 / 适配层，负责：

- 启动时 `GameManager.reset()`
- 注册 Demo NPC 初始好感
- 连接 `DayManager` 运行信号
- 将真实系统状态转成 TodayView 可消费数据
- 将真实事件转成 StoryLayer 可展示内容
- 将 `summary` 转成 DaySummaryLayer 结构
- 维护联系人、未读、当天行动记录与事件记录
- 保留测试按钮并支持拖动测试面板

## 4. 五个核心 UI Layer 当前状态

### 4.1 TodayView

当前已接通：

- 展示当天 / 当前时段
- 展示属性与资源摘要
- 展示 AP
- 展示真实手牌
- 选卡并确认后调用 `DayManager.play_card()`
- 顶部资源区已采用正式样式：精力 / 心情为条形资源展示，金钱为图标 + 数值展示
- 精力与心情数字已叠入条内，显示为 `current/max`
- 资源变化仅动画 `ProgressBar.value`，不通过改控件尺寸表现进度，避免挤动其他 UI
- 资源条底槽已收敛为深色细边框样式，精力 / 心情填充色向图标本身的暖色调对齐
- 资源区当前已改为局部自由布局：`ResourceModule` 保持模块化，`ResourceGroup` 及内部的 `EnergyInd` / `MoodInd` / `MoneyInd` 都改为普通 `Control`
- 精力 / 心情 / 金钱三组中的图标、条、数字现在都可在编辑器中单独拖动位置与微调尺寸，不再受资源区内部 `HBoxContainer` 排版限制
- 四维属性区当前已改为局部自由布局：`AttrModule` 保持模块化，`AttrGroup` 内部的四个属性项改为普通 `Control`
- 学识图标与其余三个属性点、对应数值标签现在都可在编辑器中单独拖动位置与手动放大，不再受四维区内部 `HBoxContainer` 排版限制
- `today_view_v2.gd` 不再强制把四维属性点重置为 `12 × 12`，因此手工调整尺寸可直接保留
- `today_view_v2.gd` 已加 `@tool` 编辑器样式预览支持，资源条样式可在编辑器中更早看到
- 顶部功能按钮区已落地为双入口结构：左侧电脑、右侧手机
- 手机按钮已接入现有主控链，可直接打开 `PhoneLayer`
- 电脑按钮当前使用已接入图标，定位为待开发入口，暂不接入新页面
- 功能按钮区当前也已改为局部自由布局：`FuncModule` 保持模块化，`FuncBtns` 改为普通 `Control`，电脑 / 手机按钮可在编辑器中直接拖动微调
- `UnreadDot` 仍保留在 `PhoneBtn` 下，可继续单独微调位置
- 功能按钮区保留独立图标替换空间，后续可直接换贴图，不必改 TodayView 主结构

当前定位：

- 已是主计划层入口

### 4.2 StoryLayer

当前已接通：

- 事件旁白 / 对话展示
- 事件选项展示
- 打字机效果
- 空格推进
- 回车推进
- 鼠标左键推进

当前交互规则：

- 正在打字时，输入一次会直接显示全文
- 打字结束后，再输入一次进入下一句
- 出现选项时不会误推进

### 4.3 DiceLayer

当前已接通：

- 掷骰层完整演出流程
- 1~6 骰子素材显示
- 结果文本与计算展示

当前说明：

- 现在已不是纯文本数字骰子
- 后续仍可继续把检定玩法更深地并入事件主链

### 4.4 PhoneLayer

当前已接通：

- 联系人列表
- 聊天页切换
- 回复选项
- 未读标记
- `Esc` 返回 / 关闭
- 点击手机外空白关闭

当前交互规则：

- 若在聊天页按 `Esc`，先返回联系人列表
- 若在联系人列表按 `Esc`，直接关闭手机
- 点击黑色空白遮罩，直接关闭手机

当前说明：

- 联系人与消息仍是“半真数据接线”模式
- 已可用于当前 Demo UI 流程，但还不是最终长期消息系统

### 4.5 DaySummaryLayer

当前已接通：

- 真实天数显示
- 当天行动记录展示
- 当天事件记录展示
- 属性 / 资源变化展示
- 关系变化展示
- 下一天推进按钮

当前定位：

- 已可承接 `DayManager` 的日结算输出

## 7. 当前信息架构口径

后续所有 UI 相关文档，统一按下面这套信息架构描述：

### 7.1 顶部状态栏

建议承载：
- 精力
- 心情
- 金钱
- 手机入口
- 电脑入口（可先保留占位）

当前 V2 落地口径补充：
- 精力：图标 + 固定宽度资源条，数字置于条内中央
- 心情：图标 + 固定宽度资源条，数字置于条内中央
- 金钱：图标 + 数值，不使用资源条
- 顶部功能入口当前位于状态栏右侧，顺序为左电脑、右手机
- 手机入口已接现有 `PhoneLayer` 主链，可继续沿用当前交互
- 电脑入口当前为待开发状态，但已接入正式图标素材，便于后续直接替换或扩展
- 顶部状态栏当前已拆为 `NameModule` / `ResourceModule` / `AttrModule` / `FuncModule` 四个模块根节点，可直接在编辑器中移动模块整体位置
- 资源区当前已进一步改为局部自由布局：`ResourceModule/ResourceGroup` 现在是普通 `Control` 结构，`EnergyInd` / `MoodInd` / `MoneyInd` 内部的图标、条、数字都可单独拖动和微调
- 功能按钮区当前也已改为局部自由布局：`FuncModule/FuncBtns` 现在是普通 `Control` 结构，`ComputerBtn` / `PhoneBtn` 可直接拖动位置，`UnreadDot` 可继续单独细调
- 四维属性区已进一步改为局部自由布局：`AttrModule/AttrGroup` 现在是普通 `Control` 结构，`Knowledge` / `Eloquence` / `Physique` / `Insight` 内部的 `Dot` 与 `Value` 可单独拖动和放大
- 若要移动整组资源区 / 四维区 / 功能按钮区，应优先选中对应模块根节点；若要细调资源图标、资源条、资源数字、四维图标和功能按钮，可直接选中模块内部对应节点
- 当前顶部栏已形成“外层模块可整体移动 + 模块内部可局部自由摆放”的混合结构

### 7.2 中景舞台区

建议承载：
- 校园 / 宿舍 / 教室 / 操场等背景
- 人物站位
- 一句环境提示或轻反馈
- 轻剧情氛围

### 7.3 时段信息区

建议承载：
- 当前是上午 / 下午 / 晚间
- 本时段剩余 AP
- 当前状态提示

### 7.3.1 顶部资源区与功能按钮区编辑说明

当前顶部资源区与功能按钮区调法应统一按以下原则理解：

- 顶部状态栏外层主框架仍保留模块化结构，便于整体移动与后续继续扩展名字区 / 其他入口
- `ResourceGroup` 与 `FuncBtns` 现在都已改为普通 `Control`，不再是必须依赖 `Container` 排版的内部结构
- 精力 / 心情 / 金钱三组中的图标、条、数字现在都可像四维区一样直接拖动位置
- 电脑 / 手机按钮以及 `UnreadDot` 现在也都可直接在编辑器中继续微调位置
- 资源条的业务表现仍沿用现有脚本：数值更新只动画 `ProgressBar.value`，不会因为局部自由布局而改成挤动其他 UI
- 若后续继续细调顶部栏，建议维持“外层模块负责整体布局，模块内部节点负责精修位置”的方式迭代

### 7.4 底部手牌区

建议承载：
- 本时段可选卡牌
- 卡牌取舍
- 确认执行

### 7.5 覆盖层

当进入强剧情、骰子、手机或日结算时，再切入对应覆盖层。

## 9. 当前素材风格要求

当前 Demo 的视觉风格建议保持：

- 校园青春感
- 半真实、半戏剧
- 生活感和记忆感优先
- 不做过度夸张、纯炫技式二次元表现
- 让画面服务关系、事件与氛围

### 9.1 场景图

优先级最高的是：

- 宿舍白天 / 夜晚
- 校门口
- 教学楼外
- 食堂
- 军训操场
- 校园夜路
- 社团活动点

### 9.2 卡牌图

要求：

- 一眼能认出动作类型
- 单主体清晰
- 细节不要过碎
- 缩小后仍能读图

### 9.3 头像与联系人图

要求：

- 小尺寸仍有辨识度
- 脸部特征优先
- 不同角色之间轮廓差异清晰

## 10. UI 文档表述规范

今后提到 UI 状态时，统一使用下面三类词：

### 可写“已实现”的
- V2 默认玩家入口
- 五层 UI 的当前接线状态
- 测试面板可拖动
- 剧情推进交互
- 手机关闭交互
- 骰子素材表现已接入

### 只能写“可继续增强 / 尚未最终定稿”的
- 手机长期真实消息系统
- 骰子玩法与主事件流程的更深整合
- 正式剧情资源、立绘资源、头像资源

### 必须避免的
- “默认入口还是 main.tscn / test_scene.tscn”
- “ui_test_v2 只是静态展示稿”
- “手机没有快捷关闭方式”
- “剧情只能键盘推进”

---

## 11. 当前 UI 侧下一步重点

如果后续继续打磨 UI，建议顺序是：

1. 继续补剧情文案与人物资源
2. 把 PhoneLayer 从半真数据推进到更真实的消息系统
3. 把 Dice 判定更自然地嵌回事件流程
4. 在不破坏回归测试链的前提下继续细化 V2 主链表现

这样能保持你每天直接启动可玩，同时不丢回退路径和验证通道。
