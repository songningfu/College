# 《College》项目概览与 Demo 范围

> **文档定位**：说明《College》是什么、当前 Demo 要证明什么、这次开发范围收束到哪里。  
> **使用方式**：给自己、协作者与未来版本统一项目口径；不承担程序细节说明。  
> **最后整理**：2026-04-08

---

## 1. 项目是什么

《College》是一款以**关系与成长**为主轴的大学人生模拟游戏。

它不是纯恋爱游戏，也不是纯生存管理器，而是借由日常安排、关系推进、事件演绎与回望式叙述，让玩家重新经历一次大学生活：

- 重新选择
- 重新靠近
- 重新错过
- 重新理解当年的自己

这款游戏的核心，不是“爽快通关”，而是让玩家在推进过程中慢慢产生一种感觉：

**这像真正活过的大学。**

---

## 2. 完整产品范围

《College》的完整产品目标不是一个短篇校园恋爱段落，而是：

**覆盖大学四年。**

完整版本会通过：

- 时间推进
- 关系成长
- 长期事件
- 学业与生活选择
- 大学不同阶段的情绪变化

去呈现一整段大学人生，而不是只做某一学期、某一条恋爱线、某几段记忆切片。

---

## 3. 当前 Demo 的定位

当前阶段制作的不是“大而全的前期试玩”，而是：

**首个可对外展示的高完成度 Demo 样片。**

它要验证的不是内容量，而是下面这些核心判断：

1. 大学氛围是否成立
2. 发牌式行动卡循环是否成立
3. 关系推进是否能带出持续动力
4. UI 是否能同时承载氛围、操作与轻演出
5. 剧情与系统能不能自然地接在一起

也就是说，这个 Demo 的目标不是“证明游戏很大”，而是“证明这个项目的核心体验是真的站得住”。

---

## 4. 当前 Demo 范围

### 4.1 时间范围

当前 Demo 只覆盖：

**开学前几周。**

不是完整大一上学期，也不是完整大学生活。

### 4.2 当前体验收束点

当前 Demo 的阶段性收束点仍是：

**与第一个女生完成第一次约会后结束。**

### 4.3 当前 Demo 主线

当前 Demo 应形成一条完整但不过分发散的体验弧线：

**建角 → 入学 → 固定 208 宿舍初识 → 军训与适应 → 班级 / 社团 / 手机关系推进 → 第一次约会 → Demo 结束**

这条弧线必须让玩家感到：

- 自己真的进入了大学
- 人际关系是被一天天养出来的
- 选择会慢慢长出情感结果
- 第一次约会不是突兀奖励，而是此前日常积累的回响

### 4.4 当前 Day 1 口径

当前 Demo 的 Day 1 不再使用“舍友抽取”作为体验核心，而是使用固定 208 宿舍阵容建立第一天关系起点。

当前固定口径：
- 208 舍友固定为：林逸枫 / 周文 / 陈向星 / 沈砚麒
- 开场先用黑屏居中文字建立报到日氛围
- 上午应保留报到相关卡牌与首日行动选择
- 中午和晚上再进入更强的宿舍事件与骰子演出

---

## 5. 当前体验重点

当前 Demo 的体验重点不是铺大量内容，而是打磨以下几件事：

### 5.1 日常安排感

玩家每天都要面对有限时段、有限 AP、有限精力，在发到手里的行动卡中做取舍。

### 5.2 大学关系感

舍友、同学、学长学姐、女生关系，不应只是数值条，而要让玩家逐渐感到：

- 谁更容易靠近
- 谁更让人自卑
- 谁会留下记忆
- 谁会真的影响这段大学生活的温度

### 5.3 演绎感

事件、对白、手机消息、轻检定与结算，不是额外装饰，而是把“做了什么”变成“经历了什么”。

### 5.4 回望感

《College》始终带一点作者回望意味。

它要让玩家觉得：

- 大学不仅是任务系统
- 也是关系史、成长史、遗憾史
- 某些东西当年没有重新选，这次终于有机会重来

---

## 6. 主角与叙事底色

当前主角不是完全空白的容器，而是带有明显底色的“你”：

- 刚进入大学的新生
- 带有一点“小镇做题家”气质
- 相信大学能改变人生
- 同时又隐约担心自己并不真正属于这里

这种底色不会被粗暴写成固定模板，但会渗透在：

- 口吻
- 观察方式
- 关系里的迟疑与比较
- 对“被认可”的在意

这也是《College》和一般校园恋爱游戏之间的重要差异。

## 8. 当前开发阶段判断

截至 2026-04-08，项目更准确的阶段判断是：

**核心循环已可验证，V2 UI 已成为默认入口，当前重点是把 Day 1 收束成真正可玩的首日样片。**

具体来说：

- 核心系统已能跑通 21 天测试
- 默认入口已切到 `ui_test_v2.tscn`
- Day 1 的黑屏开场、208 宿舍事件链与中晚骰子演出已接入
- 但上午卡牌阶段、事件与卡牌节奏衔接、对白与旁白边界仍在最后修正

---

## 9. 当前版本最重要的目标

当前版本最重要的不是“功能数量继续膨胀”，而是：

1. 把核心循环、关系推进和事件演绎真正拧成一条体验链
2. 把 UI 做到能稳定承载这条体验链
3. 让 Demo 在短范围内建立足够明确的大学气质与情感记忆点
4. 先把 Day 1 做成一整天能完整走通的可玩样片

如果这几件事成立，《College》后续扩到更长周期才有意义。
