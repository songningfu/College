> **原则**：先把 Demo 的主体验链拧紧，再扩系统与内容。  
> **最后整理**：2026-04-07

---

## 1. 当前阶段判断

截至目前，`College` 已经完成的不是完整 Demo，而是：

- 核心系统可运行
- 21 天测试链已建立
- UI 有骨架与原型
- 剧本方向已明确

当前最关键的问题已经从“有没有系统”转为：

**现有系统、UI、剧情能不能真正接成一条可演示的体验链。**

## 3. Demo 制作主线

## P1：把 Day 1 首日体验真正收束成可玩样片

### 目标
让 Day 1 从“能触发事件”变成“事件、卡牌、骰子、手机、总结能顺着走完的一整天”。

### 当前重点问题
- [x] 修正早晨高优先级事件直接吞掉白天时段的问题
- [x] 让 Day 1 上午正常出现报到相关手牌，不再是纯事件连播
- [ ] 继续整理“旁白文本”和“角色对白文本”的写法边界
- [ ] 校正中午枫 → 星 → 父亲提问 → 骰子，以及晚上全员到齐 → 骰子 → 收束的实际运行节奏
- [ ] 验证 Day 1 的手机反馈与日总结是否真正引用当天发生的内容
- [ ] 继续确认 V2 默认入口里的 StoryLayer / DiceLayer / TodayView 交接是否完全稳定

### 当前验收标准
- 黑屏开场稳定出现
- 上午可进入卡牌选择，不再直接跳到晚上
- 中午与晚间骰子分支都能正常落回事件链
- Day 1 对话框里的人名与正文类型一致，不再出现“挂人名但仍是旁白”的主要错位
- Day 1 能完整走到手机与日总结

### 目标
让当前 Demo 的数值、节奏和卡牌差异真正成立。

- [ ] 检查军训期是否过于压抑或过于松弛
- [ ] 检查 Day 20 顾遥好感是否稳定接近目标区间
- [ ] 检查每时段是否仍保有随机差异
- [ ] 调整保底卡、随机卡和关系增幅的节奏

### 重点观察项

## 9. 当前明确不该做的事

为了避免项目继续发散，现阶段不建议优先投入：

- 一口气铺完整四年文本
- 再造一套新的大系统替代现有 core
- 在 UI 主链未定前做大规模美术返工
- 过早细化电脑系统
- 给尚未接通的功能写大量最终版文案

当前真正重要的是“收束”，不是“继续发散”。

---

## 10. 当前阶段的完成标准

当下面这些条件成立时，可以认为 Demo 进入下一阶段：

1. 有一个正式可进入的手动体验场景
2. 一整天循环可以在 UI 中完整走完
3. 报到 / 军训 / 班级 / 社团 / 女生推进这条主线已有可演示文本
4. 约会收束点可以跑通
5. README 与 docs 已经形成单一事实入口

这之后，再去扩大系统和内容，效率才会更高。
